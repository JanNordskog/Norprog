Nettsidehastighet og
Core Web Vitals forklart

Google bruker tre konkrete ytelsesmål — LCP, INP og CLS — for å vurdere brukeropplevelsen på nettsiden din. Disse påvirker rangeringen din direkte. Her er hva de betyr, og hva du kan gjøre med det.

Hva er Core Web Vitals?

Core Web Vitals er Googles initiativ for å etablere felles kvalitetsstandarder for brukeropplevelse på nettet. I stedet for å måle hundrevis av tekniske parametere, fokuserer Google på tre metrikker som faktisk reflekterer det brukerne opplever.

"Å optimalisere for kvaliteten på brukeropplevelsen er nøkkelen til langsiktig suksess for ethvert nettsted."

— web.dev / Google, Web Vitals

Disse tre metrikkene inngår i Googles rankingsystem. De er ikke de eneste faktorene som teller, men Google er eksplisitt på at de brukes: "Core Web Vitals brukes av rangeringssystemene våre."

De tre metrikkene

LCP
Largest Contentful Paint
≤ 2,5 sekTid til største innholdselement er synlig
INP
Interaction to Next Paint
≤ 200 msResponstid på brukerinteraksjon
CLS
Cumulative Layout Shift
≤ 0,1Uforventet bevegelse av elementer på siden

LCP — Largest Contentful Paint

LCP måler hvor lang tid det tar fra en bruker klikker på lenken din til det største synlige innholdselementet er lastet inn — vanligvis et hero-bilde eller en stor overskrift. Google krever at dette skjer innen 2,5 sekunder.

En LCP over 4 sekunder er klassifisert som "dårlig" av Google. Det betyr at brukeren sitter og stirrer på en blank eller delvis lastet skjerm i over fire sekunder — noe de fleste forlater langt før.

Nettsidehastighet kan måles presist med gratis Google-verktøy som PageSpeed Insights og Search Console. Foto: Unsplash

INP — Interaction to Next Paint

INP måler hvor raskt siden responderer når en bruker klikker, tapper eller skriver noe. Grenseverdien er 200 millisekunder. Overstiger du 500 ms, betegner Google det som "dårlig".

En treg INP oppleves som at siden "henger" — du trykker på en knapp og ingenting skjer med det samme. Dette er spesielt merkbart på eldre telefoner og tregere nettverk.

CLS — Cumulative Layout Shift

CLS måler hvor mye innholdet på siden hopper rundt mens siden laster. Du har sikkert opplevd det: du er i ferd med å trykke på en lenke, og plutselig dukker det opp en annonse som skyver alt nedover. Du ender opp med å trykke på noe helt annet.

Google-grenseverdien er 0,1. Et høyt CLS-tall signaliserer at siden ikke er visuelt stabil og gir en frustrerende brukeropplevelse.

Slik påvirker det SEO-rangeringen

Google er tydelig på at relevans alltid veier tyngst. En side med ekstremt nyttig innhold kan rangere godt selv med middelmådige Core Web Vitals-tall. Men Google legger til:

Når flere sider tilbyr like relevant og nyttig informasjon, vil sterk sideytelse bli en avgjørende faktor. For norske bedrifter som konkurrerer med andre lokale aktører om de samme søkeordene, kan dette være forskjellen mellom plass 1 og plass 5 i søkeresultatene.

Slik sjekker du din side gratis

Gå til pagespeed.web.dev, lim inn adressen til nettsiden din og trykk analyser. Du får konkrete scorer for alle tre metrikkene — både for mobil og desktop — med forklaringer på hva som drar ned resultatet.

Vanlige årsaker til dårlige tall

  • Ikke-optimaliserte bilder — store bildefiler uten komprimering er den vanligste årsaken til treg LCP
  • Tredjepartsscripts — chat-widgets, analyseverktøy og annonsescripts kan forsinke all interaktivitet
  • Ingen bildedimensjoner i koden — fører til layout shift (høy CLS) fordi nettleseren ikke vet hvor mye plass bildet tar
  • For mange plugins (WordPress) — hvert plugin legger til JavaScript og CSS som nettleseren må laste og prosessere
  • Billig hosting — en treg server gir treg LCP uansett hva annet du gjør

Dårlige Core Web Vitals handler sjelden om én ting — det er som regel en kombinasjon av bilder, scripts og hosting. Foto: Unsplash

Oppsummering

Core Web Vitals er Googles tre målbare mål for brukeropplevelse: LCP (lastehastighet), INP (responstid) og CLS (layoutstabilitet). De inngår direkte i Googles rangeringssystemer, og dårlige tall kan koste deg posisjoner i søkeresultatene.

Det gode nyheten er at alle tre kan måles gratis med Googles egne verktøy, og forbedringer kan gjøres stegvis. Starter du med riktig teknisk fundament — optimaliserte bilder, rask hosting og ren kode — er mye allerede løst fra dag én.
